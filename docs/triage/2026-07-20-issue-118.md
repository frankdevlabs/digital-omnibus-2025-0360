## Assessment

**T2-10 — not a site-wide block; the *trailing slash* is the banned URL. Fixable.**
Reproduced live on 2026-07-20:

| URL | Result |
|---|---|
| `https://ieu-monitoring.com/feed/` | **503** + Wordfence block page, `Block Reason: Accessed a banned URL` |
| `https://ieu-monitoring.com/feed` (no trailing slash) | **200**, `content-type: application/rss+xml`, 626 KB, 50 `<item>`s, `lastBuildDate` Sat 18 Jul 2026 |

The block is a Wordfence URL rule on the slash variant, not an IP ban and not a bot check:
the no-slash variant returns 200 with a plain `curl` and **no** User-Agent spoofing, repeatably
(3/3 consecutive requests), while the slash variant 503s in the same session. Other candidate
endpoints (`/?feed=rss2`, `/rss`, `/category/eu-council/feed/`) all 503 — so the no-slash
canonical feed is the one working entry point.

T2-10 matters more than a normal Tier-2 stakeholder feed: it is the documented workaround for
Consilium's bot-block (Telecom/TTE Council main results = Tier-1-grade signal). Confirmed today
that consilium.europa.eu still refuses the tracker directly (`/en/meetings/tte/` and
`/en/press/press-releases/` → **403**), so the workaround cannot simply be dropped. Fix the URL
rather than waiting out the 14-day pause threshold.

Content check: the 50 current items carry 0 `digital omnibus` / `2025/0360` / `ePrivacy` matches —
no missed hit during the two blocked runs.
